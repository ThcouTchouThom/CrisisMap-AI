# CrisisMap AI

Academic prototype for analyzing pre- and post-disaster satellite image pairs from the xBD/xView2 dataset and producing a visual damage map.

## Project Structure

```text
CrisisMap AI/
  app/                         # Prototype UI or demo scripts.
  configs/                     # Experiment and path configuration files.
  data/
    raw/                       # Extract the official xBD/xView2 archive here.
    processed/                 # Derived tensors, chips, masks, or metadata.
    samples/                   # Tiny hand-picked samples for demos/tests only.
  notebooks/                   # Exploratory analysis and experiment notebooks.
  outputs/
    checkpoints/               # Model weights created during training.
    figures/                   # Plots and visual summaries.
    predictions/               # Generated damage maps and prediction artifacts.
  src/crisismap/
    data/                      # Dataset loading and preprocessing code.
    evaluation/                # Metrics and validation utilities.
    models/                    # Model architectures.
    training/                  # Training loops and experiment runners.
    visualization/             # Damage map rendering utilities.
```

## Dataset Placement

Do not commit the xBD/xView2 dataset or generated model artifacts to version control.

When the dataset is available locally, extract the official xBD/xView2 archive into:

```text
data/raw/xbd/
```

For example, the raw dataset files should live under `data/raw/xbd/` after extraction, while any resized images, masks, labels, or training-ready arrays generated by this project should be written to `data/processed/`.

## First Steps

1. Create and activate a Python virtual environment.
2. Install dependencies with `pip install -r requirements.txt`.
3. Place the extracted xBD/xView2 archive under `data/raw/xbd/`.
4. Keep only small demonstration assets in `data/samples/`.
