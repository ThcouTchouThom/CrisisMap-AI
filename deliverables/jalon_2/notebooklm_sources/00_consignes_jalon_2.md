# Consignes jalon 2 - 8INF934

Le jalon 2 demande une démonstration de l'avancement technique du projet. Pour CrisisMap AI / Aftermath, le livrable doit rester centré sur les éléments suivants :

- données accessibles et chargées ;
- visualisation d'exemples du dataset ;
- séparation train / validation / test ;
- première exploration du dataset ;
- baseline IA fonctionnel ;
- premières métriques quantitatives ;
- organisation initiale du dépôt Git ;
- présentation de 6 à 8 minutes avec exemples de données, baseline et architecture prototype.

Le projet est déjà plus avancé que ce niveau minimal. Pour la présentation, il faut donc distinguer :

- le socle attendu pour le jalon ;
- les travaux avancés qui démontrent la maturité du projet ;
- les résultats encore expérimentaux, à ne pas présenter comme définitifs.

Message clé : le projet dispose d'un pipeline complet de segmentation des dommages, depuis les données xBD/xView2 jusqu'à un prototype Streamlit, avec une attention particulière à la qualité méthodologique des splits.
